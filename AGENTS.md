# Posoco Agent Guide

This is a [MoonBit](https://docs.moonbitlang.com) project: **`colmugx/posoco`** — a 0-dependency LLM agent framework using hexagonal (ports-and-adapters) architecture.

## One-Line Summary

Posoco defines 7 traits + an Agent struct with a `run_turn` loop; no external dependencies, no generics, uses MoonBit `&Trait` references throughout.

## Architecture

```
Layer 3 — Agent (program/composition)
  Agent struct holds &Trait references; run_turn() orchestrates the loop

Layer 2 — Interpreters (runtime/injection)
  Users provide concrete trait implementations (mocks in tests, real in prod)

Layer 1 — Traits (effect interfaces)
  ModelPort, ToolProvider, ToolRuntime, SessionStore, Observer
  + Compressor (optional), PipelineHook (optional)
```

### Source Files

| File | Purpose |
|------|---------|
| `port.mbt` | All types (Message, ToolCall, etc.) + 7 pub(open) traits |
| `agent.mbt` | Agent struct + AgentConfig + run_turn loop |
| `builtin.mbt` | NoopCompressor, NoopHook, HookChain, helper functions |
| `posoco_test.mbt` | 9 blackbox tests (uses @posoco prefix) |
| `posoco_wbtest.mbt` | 12 whitebox tests (direct symbol access) |
| `pkg.generated.mbti` | Generated by `moon info` — commit when public API changes |

## Key Architecture Rules

- **Split your pre-commit**: The pre-commit hook only runs `moon check`. The full 4-step checklist above is for _before pushing_.
- **Two hook layers**: `Observer` (read-only, returns Unit, never blocks) vs `PipelineHook` (can mutate stages, returns Result, can abort turn).
- **Tool failure is non-fatal**: tool errors become messages for the LLM; only model errors abort the turn.
- **Compressor is pure**: returns instructions (None/Replace/NewThread) — never saves sessions or emits events directly. Agent and Observer handle side effects.
- **Compression runs every LLM round** (before each model call), not once per turn.
- **All traits are `pub(open)`** so external packages can implement them.

## MoonBit Syntax Traps (Common Mistakes)

| Trap | Wrong | Correct |
|------|-------|---------|
| Trait impl | `impl Trait for Type { fn method(...) }` | `pub impl Trait for Type with method_name(self, ...) { }` |
| Try operator | `func()?` | Match `Ok`/`Err` explicitly |
| Optional chain | `self.hook?.call()` | `match self.hook { Some(h) => ..., None => () }` |
| Map literal | `{ key: value }` | `Map::from_array([("key", value)])` |
| Trait object | `Box<dyn Trait>` | `&Trait` — Agent has no generics |
| Named enum variant | `Variant { field: Type }` | `Variant(field~ : Type)` — note `~` label |
| Infinite loop | `loop { }` (Rust) | `while true { }` |
| Struct fields `mut` | Needs `mut` on Map/Array fields | Only `mut` on scalar index counters, not on Array/Map fields |
| Map access | `map[key]` without guard | Guard with `.contains()` first (returns V, not Option[V]) |
| Json null | `Json::Null` | `Json::null()` |

## Mock Pattern for Tests

All tests use the same 4-mock setup (blackbox uses `@posoco` prefix, whitebox uses direct access):

```moonbit
let model = MockModelPort::{ responses: [stop_response("hi")], index: 0 }
let tools = MockToolProvider::{ tools: [] }
let runtime = MockToolRuntime::{ results: Map::from_array([]) }
let store = MockSessionStore::{ store: Map::from_array([]) }
let observer = MockObserver::{ events: [] }
let config = AgentConfig::{ max_tool_rounds: 10, .. }
let agent = Agent::new(model, tools, runtime, store, observer, NoopCompressor::{ }, None, config)
let result = agent.run_turn(user_msg("hello"), "session_1")
```

Key mock conventions:
- `MockModelPort` — `mut index : Int`, cycles through `responses` array, fails when exhausted
- `MockToolRuntime` — looks up tool by name in `results` Map; guard with `.contains()` first
- `MockSessionStore` — in-memory map; returns empty session for unknown IDs
- `MockObserver` — records all events for post-turn inspection
- Always use `Map::from_array([])` for empty maps and session metadata
- Always use `Json::null()` for ToolCall arguments

## Extension Model

Extensions are independent MoonBit projects (e.g., `colmugx/posoco-ext-rtk`) that import `colmugx/posoco` and implement one trait. Posoco core has zero runtime deps on any extension.
